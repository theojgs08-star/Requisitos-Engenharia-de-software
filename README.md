# Requisitos-Engenharia-de-software
Eu estou realizando o levantamento de requisitos em um projeto de banco de dados, identificando quais dados o sistema precisa armazenar e como eles serão usados.  Eu estou trabalhando com banco de dados, que é a área da computação responsável por organizar, armazenar e gerenciar informações de forma estruturada...
Estou aprendendo na faculdade a disciplina de requisitos de software, que é uma das partes mais importantes no desenvolvimento de sistemas. Nela, estou entendendo como identificar, analisar e organizar tudo o que um sistema precisa fazer antes de começar a programação. Isso ajuda a evitar erros, retrabalho e garante que o sistema atenda corretamente às necessidades do usuário.

Um dos principais pontos que estudo são os requisitos funcionais, que definem o que o sistema deve fazer, como cadastrar usuários, realizar vendas, controlar estoque ou gerar relatórios. Também estudo os requisitos não funcionais, que estão relacionados à qualidade do sistema, como desempenho, segurança, usabilidade e confiabilidade. Esses conceitos são fundamentais para pensar em sistemas mais profissionais e bem estruturados.

Além disso, estou aprendendo sobre a importância da documentação de requisitos, onde tudo é descrito de forma clara para que desenvolvedores e clientes entendam exatamente o funcionamento do sistema. Isso inclui também casos de uso, histórias de usuário e fluxos de processo.

Outro ponto importante da disciplina é o estudo de metodologias ágeis, principalmente o Scrum. Nele, aprendo como os projetos são organizados em ciclos chamados sprints, onde pequenas entregas são feitas de forma contínua. Também entendo os papéis dentro do Scrum, como Product Owner, Scrum Master e equipe de desenvolvimento, além de práticas como reuniões diárias, planejamento de sprint e revisões.

Essa matéria me ajuda a entender como funciona o desenvolvimento de software no mercado real, onde não se programa apenas, mas se planeja, organiza e evolui o sistema constantemente com base nas necessidades do cliente. Também me ajuda a conectar a lógica de programação com o desenvolvimento de sistemas completos, como os projetos que estou criando em Python, bancos de dados e sistemas de gestão.

No geral, estou aprendendo a pensar como um desenvolvedor profissional, entendendo não só o código, mas todo o processo de construção de um sistema, desde a ideia inicial até a entrega final.
